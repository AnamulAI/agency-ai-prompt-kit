# Client Discovery Prompt

Use this prompt to understand a client project before writing copy, offers, strategy, or landing pages.

## Prompt

You are helping me structure a client discovery summary.

### Input
- Service Name: [Insert service name]
- Target Audience: [Insert audience]
- Industry/Niche: [Insert niche]
- Main Problem They Face: [Insert problem]
- Desired Outcome: [Insert result]
- Timeline: [Insert timeline]
- Known Constraints: [Insert constraints]
- Current Assets They Have: [Insert assets]

### Task
Create a structured client discovery summary with these sections:

1. Business context  
2. Audience insight  
3. Core pain points  
4. Desired transformation  
5. Offer opportunity  
6. Messaging angle  
7. Suggested next steps  

### Output Rules
- Keep it practical and clear
- Make it useful for real service delivery
- Avoid generic business filler
- Focus on decisions that improve execution

## Example Use Case

- Service Name: Website SEO Optimization
- Target Audience: service businesses
- Industry/Niche: local service
- Main Problem They Face: low visibility and weak lead flow
- Desired Outcome: better rankings and more qualified leads
- Timeline: 30 days
- Known Constraints: limited content and weak website structure
- Current Assets They Have: website, Google Business Profile, a few service pages

## Notes

This prompt works well for:
- discovery calls
- service scoping
- offer positioning
- landing page planning
- client onboarding
